3.2. Defining overlays

Okay. Here we see a definition and example of an overlay:

Overlays are Nix functions which accept two arguments, conventionally called self and super, and return a set of packages. For example, the following is a valid overlay.

self: super:

{
  boost = super.boost.override {
    python = self.python3;
  };
  rr = super.callPackage ./pkgs/rr {
    stdenv = self.stdenv_32bit;
  };
}

Alright. I can sort of see what we‚Äôre doing here. The manual explains that self is the ‚Äúfinal‚Äù package set (I assume this means the package set with all of the overlays applied), and super is the ‚Äúcurrent‚Äù state, if we think of overlays as folding over a set of packages. Overlays are similar to other methods for customizing Nixpkgs, in particular the packageOverrides attribute described in Section 2.5, ‚ÄúModify packages via packageOverrides‚Äù.
